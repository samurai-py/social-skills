# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repository is

A **marketplace of Claude Code plugins** for marketing/content work, organized as a vertical
slice per brand, plus **[canva-killer](canva-killer/)** — an AI-drivable social art generator
(templates as code instead of proprietary Canva files) — and **[identity/](identity/)**, the
visual identity of the SocialSkills product itself.

```
plugins/marketing-tools/   # generic, brand-agnostic skills (SEO/Ads) — public
canva-killer/                # art generator: core + studio + base templates — public
identity/                    # SocialSkills' own visual identity (mascot, mark, rules) — public
_templates/                   # skeletons for scaffolding a new brand/channel/template
user/                          # YOUR data: real brands, brand plugins, exported art — private, gitignored
```

Framework (public, versioned) = `plugins/marketing-tools/`, `canva-killer/{src, studio, base
templates, _TEMPLATE}`, `identity/`, `_templates/`. Everything specific to whoever runs this repo
— your brand's voice/channels, real palette/logo, brand-authored templates, exported PNGs — is
isolated in `user/` (see [`user/` convention](#user-convention) below). Never put real brand data
directly under `plugins/` or `canva-killer/`; it belongs in `user/`.

## marketing-tools skills

Generic, brand-agnostic SEO/Ads skills. Invoke as `/marketing-tools:<skill>`.

| Skill | Command | Needs an API |
|-------|---------|----------------|
| Brand voice interview | `/marketing-tools:brand-voice` | No |
| Gap zone (keywords rank 5–20) | `/marketing-tools:gap-zone` | GSC (AdsAgent) |
| Competitor analysis | `/marketing-tools:competitor-analysis` | Apify |
| Content writer | `/marketing-tools:content-writer` | No* |
| Weekly rank tracker | `/marketing-tools:rank-tracker` | GSC (AdsAgent) |
| AI shopping optimization | `/marketing-tools:ai-shopping` | No |
| Static ads generator (Meta Ads, FAL AI + Gemini) | `/marketing-tools:static-ads` | FAL AI, Gemini |
| Google Ads audit | `/marketing-tools:ads-audit` | Google Ads (AdsAgent) |
| Wasted spend finder | `/marketing-tools:waste-finder` | Google Ads (AdsAgent) |

*`content-writer` requires a `brand-profile.json` generated by `brand-voice`.

### First-time recommended flow

1. `/marketing-tools:brand-voice` — creates `brand-profile.json`
2. `/marketing-tools:gap-zone <your-domain>` — finds opportunities
3. `/marketing-tools:competitor-analysis <keyword>` — understands the competition
4. `/marketing-tools:content-writer <keyword>` — writes the article
5. Repeat weekly with `/marketing-tools:rank-tracker <your-domain>`

## Brand plugins (vertical slices)

Each real brand is its own plugin under `user/plugins/<brand>/`: the voice lives in the slice,
the mechanics (how to post, format, length limits) live in `shared/` skills reused across
channels. Registered in the root `.claude-plugin/marketplace.json` (`source:
"./user/plugins/<brand>"`). Brand plugins are **never committed** — they're example entries in
`user/`, gitignored.

## canva-killer

See the [canva-killer README](canva-killer/README.md) for the full guide. In short: templates
are code (HTML/CSS/SVG), driven by an AI agent through MCP tools (`render`, `render_carousel`)
instead of Canva's Enterprise-only autofill. Has a local **Studio** (`npm run studio`,
`canva-killer/studio/`) with three tabs:

- **Compose** — semantic layer. Picks a ready template, generates a form from its
  `{{tokens}}`, fills content only. This is what an AI agent drives to turn data into a PNG.
- **Create/edit** — foundational layer. Builds the *layout* itself: blocks (title, kicker, text,
  image, retro window, color panel) dragged with snap-to-grid, saved as `.html` with the block
  model embedded. Supports keyboard control (arrows move, `[`/`]` reorder z-stack, Del deletes,
  Ctrl+D duplicates, Esc deselects) and per-block front/back buttons.
- **Brand** — palette/fonts/logo editor with live preview; saves to `user/canva-killer/brands/`.

The same `.html` feeds both Compose and Create/edit — `canva-killer/src/converter.mjs` bridges
block-model ⇄ HTML in code, so there's a single source of truth. Block order in the array = stack
order in both the editor and the saved HTML.

## identity/

SocialSkills (the platform, not the brands it renders for) has its own visual identity, documented
in [`identity/README.md`](identity/README.md): the UI has no fixed color — accents cycle the RGB
spectrum when no brand is loaded, and lock to the loaded brand's accent color otherwise (same rule
for the mark, the terminal dots, the hover sweep on buttons, and the chameleon mascot). The mascot
is a **companion** perched on the header, not part of the logo lockup — clicking it cycles through
body variants (chibi is default). Sprites are generated from ASCII maps by
`identity/gen-mascotes.mjs` (1 character = 1 pixel); edit the map and rerun to tweak a sprite.
The studio applies these rules — see `canva-killer/studio/index.html`.

## `user/` convention

Everything specific to whoever runs this repo — user data or generated output — lives in `user/`
(already in `.gitignore`, never committed):

```
user/plugins/<brand>/           # real brand plugins (voice, channels)
user/canva-killer/brands/       # real palette/fonts/logo per brand
user/canva-killer/assets/       # authored SVGs (logo, custom icons)
user/canva-killer/templates/    # brand-specific authored templates
user/canva-killer/out/          # exported PNGs
```

`canva-killer` resolves brand/template/custom-icon by **overlay**: it looks in
`user/canva-killer/` first, then falls back to the framework underneath. This lets the same
checkout serve as both the public template (just the `_TEMPLATE`/example) and a private install
(your real data), without duplicating code or leaking real content into git history.

To create your own brand or channel, follow [`_templates/README.md`](_templates/README.md).

## Configuration

Edit `.claude-plugin/marketplace.json` to register your brand plugins (`source:
"./user/plugins/<brand>"`). `.mcp.json` files pointing to MCP servers outside this repo use
environment variables — export them in your shell:

```bash
export CONTENT_MCP_PATH=/path/to/your/content-mcp/checkout
export STATIC_ADS_MCP_PATH=/path/to/your/static-ads-mcp/checkout
```

API keys (AdsAgent, Apify, FAL AI, Gemini) live in `plugins/marketing-tools/.mcp.json` as
placeholders — replace with your own. Brand secrets (e.g. a Strapi token) live in `.env` inside
that brand's plugin under `user/` — never committed (copy the matching `.env.example`).

## Files generated by skills

- `brand-profile.json` — brand voice profile (generated by `brand-voice`)
- `rank-history/[date].json` — ranking history (generated by `rank-tracker`)
- `ads-audit-[date].json` — audit history (generated by `ads-audit`)
